# DESIGN.md — 設計の意図とルール

このドキュメントは「**なぜそう決めたか**」を記録する。トークンの**値そのもの**は `globals.css` が正（source of truth）であり、ここでは値の背景・使い方・禁止事項を定める。

- `globals.css` … 値（何を）— color / spacing / radius / shadow の実体
- `docs/DESIGN.md`（本書）… 意図とルール（なぜ・どう使うか・何をしないか）

迷ったら、本書の「3語」と「やらないこと」に照合する。両者と矛盾する案は採用しない。

---

## 1. ブランドの3語（北極星）

すべての判断はこの3語に照合する。3語"すべて"に当てはまらない要素は却下する。

| 語 | 意味 | 画面への落とし方 |
|---|---|---|
| **信頼できる** | 業務として安心して任せられる | 青CTA、構造的で整然としたレイアウト、純黒でない落ち着いた文字色 |
| **親しみやすい** | 堅すぎず人懐っこい | ウォームグレー基調、Noto Sans JP、角張りすぎない柔らかさ |
| **クリエイターらしい** | 人柄が立ち上がる | エディトリアルな余白、コンテンツ前面、ダークサイドバーのクロム感 |

## 2. カラー

### レイアウト構造と色の関係

```
┌─────────────────────────────────────┐  bg-sidebar (#1c1a19) — ウォームダーク
│  ┌──────┐  ┌─────────────────────┐  │
│  │ side │  │  bg-background      │  │  #F2F1EF — ウォームオフホワイト
│  │  bar │  │  ┌───────────────┐  │  │
│  │      │  │  │  bg-card #fff │  │  │  白カード — コンテンツの舞台
│  │      │  │  └───────────────┘  │  │
│  └──────┘  └─────────────────────┘  │
└─────────────────────────────────────┘
```

- **ダークサイドバー（`--sidebar`）**：ウォームニアブラック。クロム感を出してコンテンツを際立たせる役割。
- **ウォームオフホワイト（`--background`）**：カードの下地。白すぎず、サイドバーとのコントラストで浮き上がる。
- **白カード（`--card`）**：コンテンツが乗る舞台。背景との重なりで奥行きを作る。

### 青CTA（`--cta`）
- 「信頼できる」の核。**新規作成など行動喚起ボタン**に使う唯一の強調色。
- `bg-cta / text-cta-foreground / hover:bg-cta-hover` のセットで使う。
- **ボタン以外への拡張は慎重に**。タブ下線・アイコン・リンク等に青を乱用するとCTAが埋もれる。

### ウォーム基調
- 背景・カード・ボーダー・補助テキスト・影まで**暖色側に寄せる**。1トークンだけ温めてもクールに転ぶ。
- 本文文字色は純黒でなく暖色ニアブラック（`--foreground: #1c1a19`）。影も暖色ティント（`rgba(28,25,23,…)` 系）。
- 青CTAはポイントとして成立するが、**青を増やすほどウォーム感が薄れる**ことを意識する。

### リンク／人柄の色（`--link`）
- リンク・選択状態など**"ここぞ"だけ**に使う personality 色。バーントシエナ系の暖色。
- 面で使わない（大きな背景・カード全体への適用は避ける）。

### 分類バッジの色（販売形態チップ）
- 4種の販売形態を色で区別する。セマンティックトークンの `/10` 透過背景 ＋ トークン色テキストで統一。
- `border-xxx/25 bg-xxx/10 text-xxx` のパターンで実装する。

| 形態 | トークン | 色調 |
|---|---|---|
| デジタル | `--cta`（青） | 青 |
| コース | `--success`（ティール） | 緑系 |
| 予約 | `--warning`（アンバー） | 橙系 |
| サブスク | `--link`（シエナ） | 赤橙系 |

- バッジ色は**分類専用**。他の用途（状態・エラー・強調）に同じトークンを使う際は文脈で判断する。
- 新しい販売形態を追加する場合は既存4色と重複しない semantic token を当てること。

### `--accent` はブランド色ではない（shadcn 予約）
- shadcn コンポーネントが hover/active **背景**に内部使用する予約トークン。ここにブランド色を入れない。
- 中身はウォームな控えめ bg（surface ランプの一段）にする。

> 具体的な hex 値は `globals.css` を参照。本書では値を二重管理しない。

## 3. インタラクション状態

ユーザーが「選んでいる / 入力している」状態の色は **`--cta`（青）で統一**する。黒（`--primary`）はCTAボタン専用ではなく、トグル選択など補助的な用途にも使う。

### フォームコントロールの選択・フォーカス

| コンポーネント | 選択/ON 状態 | フォーカス状態 |
|---|---|---|
| **Checkbox** | `bg-cta text-cta-foreground` | `border-cta ring-cta/30` |
| **Switch** | `bg-cta`（トラック色） | `border-cta ring-cta/30` |
| **Input / Textarea** | — | `border-cta`（ring なし） |
| **ToggleGroup item** | `bg-sidebar text-sidebar-primary` | デフォルト |

### ルール

- **「選んでいる」= 青（`--cta`）**：Checkbox・Switch のON状態はすべて `bg-cta`。
- **「フォーカス中」= 青ボーダーのみ**：Input / Textarea は `focus-visible:border-cta focus-visible:ring-0`。ぼかしリング（ring-[3px]）は使わない。
- **トグルの選択状態**は青CTAと競合するため `bg-sidebar`（ウォームダーク）で区別。
- **CTA ボタン**（新規作成・保存・作成）は `bg-cta text-cta-foreground hover:bg-cta-hover` のセットで使う。

## 4. タイポグラフィ

### 決定：Noto Sans JP（和文）+ Inter（Latin）
- 「信頼・親しみ」に最も素直に効く組み合わせ。日本語UIは**和文フォントが性格の8割**を決めるため、システムフォント／`-apple-system` のままにしない。

### 実装メモ
- `next/font` で読み込む。**Latinを先・JPを後**に指定し、Latin字形はInter、JP字形はNotoに落とす。
- Noto Sans JP は巨大。使う太さだけ（**400 / 500 / 700**）に絞り、`preload: false` でフルセットのブロッキングを避ける。

### 任意のレバー：見出し明朝
- 余白だけで「クリエイターらしさ」が出きらない時、**大見出しだけ明朝/セリフ系**にするとエディトリアル感が立つ。現状は未採用（要検討）。

## 4. 余白・角丸・影

- **余白**：エディトリアルにゆったり取る。詰め込まない。余白は機能であり人柄。
- **角丸**：丸すぎない一定値。やわらかいが子供っぽくしない。全要素一律のぼってり角丸を避ける。
- **影**：分離はまず**ボーダー＋余白**で行い、影は最小限。主に `--shadow-xs` / `--shadow-sm`。**ソフトな影の乱用＝AI感**なので強くしない。

---

## 5. やらないこと（AI感を狩る）

- ❌ **青CTAの乱用**：`bg-cta` はボタン（新規作成など）のみ。タブ下線・バッジ・アイコン・リンクへの青の拡張は一つ一つ判断する。
- ❌ システムフォント／`-apple-system` のままの和文。
- ❌ 全要素一律の大きな角丸 ＋ ソフトな影の乱用（ぼってり感）。
- ❌ 商品/コンテンツのサムネに**絵文字**。実画像・アバター・モノグラムにする。
- ❌ グラデの多用、無意味なキラキラ装飾。
- ❌ ウォームトーンを壊す純粋な白・純黒の多用（`--foreground` / `--background` のウォームニアブラック・オフホワイトを使う）。

---

## 6. 参照の借り方（Mobbin等）

- 借りていいのは**骨格だけ**（情報設計・階層・要素配置・操作順序）。
- **色・書体・角丸・影・装飾は借りない**。常に本書のルールと `globals.css` のトークンで上書きする。
